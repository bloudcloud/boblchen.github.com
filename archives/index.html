stage3d_7_加载obj模型

OBJ文件是Alias|Wavefront公司为它的一套基于工作站的3D建模和动画软件”Advanced Visualizer”开发的一种标准3D模型文件格式，很适合用于3D软件模型之间的互导，也可以通过Maya读写。比如你在3dsMax或LightWave中建了一个模型，想把它调到Maya里面渲染或动画，导出OBJ文件就是一种很好的选择。目前几乎所有知名的3D软件都支持OBJ文件的读写，不过其中很多需要通过插件才能实现。OBJ文件是一种文本文件，可以直接用写字板打开进行查看和编辑修改。

OBJ文件的特点

OBJ3.0文件格式支持直线(Line)、多边形(Polygon)、表面(Surface)和自由形态曲线(Free-form Curve)。直线和多角形通过它们的点来描述，曲线和表面则根据它们的控制点和依附于曲线类型的额外信息来定义，这些信息支持规则和不规则的曲线，包括那些基于贝塞尔曲线(Bezier)、B样条(B-spline)、基数(Cardinal/Catmull-Rom)和泰勒方程(Taylor equations)的曲线。其他特点如下：

  • OBJ文件是一种3D模型文件。不包含动画、材质特性、贴图路径、动力学、粒子等信息。
  • OBJ文件主要支持多边形(Polygons)模型。虽然也支持曲线(Curves)、表面(Surfaces)、点组材质(Point Group Materials)，但Maya导出的OBJ文件并不包括这些信息。
  • OBJ文件支持三个点以上的面，这一点很有用。很多其它的模型文件格式只支持三个点的面，所以导入Maya的模型经常被三角化了，这对于我们对模型进行再加工甚为不利。
  • OBJ文件支持法线和贴图坐标。在其它软件中调整好贴图后，贴图坐标信息可以存入OBJ文件中，这样文件导入Maya后只需指定一下贴图文件路径就行了，不需要再调整贴图坐标。

OBJ文件的基本结构

OBJ文件由一行行文本组成，注释行以符号“#”为开头，空格和空行可以随意加到文件中以增加文件的可读性。有字的行都由一两个标记字母也就是关键字(Keyword)开头，关键字可以说明这一行是什么样的数据。多行可以逻辑地连接在一起表示一行，方法是在每一行最后添加一个连接符()。 注意连接符()后面不能出现空格或Tab格，否则将导致文件出错。

下列关键字可以在OBJ文件使用。在这个列表中, 关键字根据数据类型排列，每个关键字有一段简短描述。

顶点数据(Vertex data)：
v 几何体顶点(Geometric vertices)
vt 贴图坐标点(Texture vertices)
vn 顶点法线(Vertex normals)
vp 参数空格顶点 (Parameter space vertices)

自由形态曲线(Free-form curve)/表面属性(surface attributes):
deg 度(Degree)
bmat 基础矩阵(Basis matrix)
step 步尺寸(Step size)
cstype 曲线或表面类型 (Curve or surface type)

元素(Elements):
p 点(Point)
l 线(Line)
f 面(Face)
curv 曲线(Curve)
curv2 2D曲线(2D curve)
surf 表面(Surface)

自由形态曲线(Free-form curve)/表面主体陈述(surface body statements):
parm 参数值(Parameter values )
trim 外部修剪循环(Outer trimming loop)
hole 内部整修循环(Inner trimming loop)
scrv 特殊曲线(Special curve)

s 光滑组(Smoothing group)
mg 合并组(Merging group)
o 对象名称(Object name)

显示(Display)/渲染属性(render attributes):
bevel 导角插值(Bevel interpolation)
c_interp 颜色插值(Color interpolation)
d_interp 溶解插值(Dissolve interpolation)
lod 细节层次(Level of detail)
usemtl 材质名称(Material name)
mtllib 材质库(Material library)
shadow_obj 投射阴影(Shadow casting)
trace_obj 光线跟踪(Ray tracing)
ctech 曲线近似技术(Curve approximation technique)
stech 表面近似技术 (Surface approximation technique)

纹理

纹理也称为贴图。之前的demo里面给出的方块就是一个纯色的。但是在实际的开发中，模型往往不是纯色的，一般都是有各种颜色，然后就可以非常真实的模拟真实世界中的物体。例如如下英雄连萌中的亚索贴图:


其实贴图就是一张非常普通的图片。但是这个图片的内容看起来非常诡异。几乎无法分辨出图片的内容。既然贴图是一张2D的图片，那么它是如何与3D结合使用的？

# UV坐标

2D纹理需要和3D模型结果起来，就需要一组坐标来定位像素，告诉3D坐标点对应着哪一个像素，由于纹理是2D的，因此坐标也是2D的。由于图片的尺寸多种多样，例如1x1、2x2、4x4、8x8等等，为了让这个坐标适配所有的图片尺寸，采用了和NDC空间坐标类似的概念，在【正常】情况下纹理坐标(UV)从左至右为[0, 1]，从上至下为[0, 1]。这样无论图片多大，最右边的一定是1，最下边的一定是1。纹理坐标轴在图片的左上角，横向为U轴，纵向为V轴。如图所示:

我们只需要在3D模型中每一个坐标点指定一个对应的纹理坐标，就可以将整个纹理按照这个坐标将像素包裹在3D模型上面。例如之前的mode，那个正方形。如果我们在正方形的左上角顶点指定(0, 0)纹理坐标，右上角顶点指定(1, 0)坐标，左下角指定(0, 1)坐标，右下角指定(1, 1)纹理坐标。那么这个图片刚好可以映射到正方形上面，那么这个正方形的最终效果就是一个木箱的一面。如果我们使用6个这个的正方形拼凑起来，就可以拼凑出一个如下图所示的木箱模型。

也许有同学会问，为毛一个正方形就指定了四个点的纹理坐标，正方形中间是如何显示出来的。其实在我们指定了四个点的纹理坐标之后，GPU在片段着色阶段，会根据每一个三角进行着色，着色的过程中会在两个顶点之间的纹理进行插值处理。这也即是为何在之前说到片段着色程序的运行次数是顶点着色程序的几十几百倍的原因了。如下图(下图中使用的是另一个纹理坐标系，我们在遇到这种纹理坐标系时，只需要做一个(u, 1-v)的处理就可):

过滤器

倍增

纹理贴图元素应该被视为在一个连续图像上的离散颜色采样;不应该被视为矩形区域。如果当观察点离场景中的一面墙很近时,墙会被放大,以至于会盖住整个屏幕。如果显示器的分辨率为 1024×1024,墙体纹理的分辨率为256×256,那么就会出现倍增问题——我们将要用很少的纹理元素来覆盖很多的像素。在本例中,每个纹理元素要覆盖4 个像素。当顶点纹理坐标在三角形表面上进行插值时,每个像素都会得到一对唯一的纹理坐标。所以,像素的纹理坐标不会与任何一个纹理元素点对应。我们可以使用插值方法来估算纹理 元素之间的颜色。GPU提供了两种插值方法:常量插值和线性插值。线性插值是我们最常用的插值方法。

常量插值(nearest)

常量插值也称为最近点采样,因为它总是用最近的纹理元素的值作为采样结果。因此它的效果非常高，但是在会出现明显的块状。如下图:

双线性插值(bilinear)

线性插值就是在两个元素点之间进行插值。双线性插值其实就在U和V上面分别进行一次线性插值。双线性插值效果会比常量插值高出很多倍，但是性能没有常量插值高效。如下图:

缩减

缩减与倍增的情况刚好相反。在缩减中,较多的纹理元素会被映射为较少的像素。例如:我们将一幅256×256的纹理映射到墙体上。然后把观察点对准墙体,并向后移动观察点,使墙体逐渐变小,直到墙体在屏幕上只占64×64的像素区域为止。现在,我们要把256×256个纹理元素映射为64×64个屏幕像素。在这一情景中,像素的纹理坐标不会与纹理贴图中的任何一个纹理元素对应。常量和线性插 值过滤器仍然适用于缩减情况。不过,缩减的处理工作稍多一些。简单的讲,我们要把256×256个纹理元素均匀缩减为64×64个纹理元素。多级渐近贴图映射(mipmapping)为此提供了一种估值手段,但是需要额外占用一些内存。

mip

我们提供gpu不同略缩版的贴图。然后gpu去寻找不同略缩版的贴图，并进行插值进行显示。例如:前面提到的256x256纹理。如果我们提供128x128、64x64、32x32、16x16、8x8、4x4、2x2、1x1等略缩尺寸的该贴图给GPU。当GPU遇到前面提到墙体在屏幕上只占64x64的像素区域时。GPU就会直接将我们上传的64x64版本的贴图显示出来。这样就能保证我们在距离足够远时，看到的图像仍然是正常的。如下图:

插值

如果之前提到的256x256不是缩减到64x64，而且缩减到50x50。那么这个时候GPU就会进行插值处理。

常量插值

常量插值就是选择最近的略缩纹理进行插值，例如选择64x64进行插值。

线性插值

线性插值会选择最近的两个略缩图进行线性插值，例如在64x64与32x32之间进行线性插值。

各向异性过滤

在AGAL1中不支持，在AGAL2中支持。当摄像机与面的法线垂直时,图像会失真。而各向异性过滤则是缓解这种情况引起的失真问题。

投影

之前在demo中，正方形的数据如下:
meshData = Vector.([
//X, Y, Z,
-1, -1, 0, // 左下角
1, -1, 0, // 右下角
1, 1, 0, // 右上角
-1, 1, 0, // 左上角
]);
因为传递给GPU的数据已经是符合GPU NDC区间的值。但是在实际的开发中，模型的数据不可能是[-1, 1]区间的。这还只是模型的数据，如果还要做一些变换，例如左移10单位，缩放2倍，x轴旋转79°等等。那么势必会在cpu里面进行计算，然后在传递给GPU。但是模型的数据往往非常庞大，所以这个过程是非常不合理的。查阅AGAL的指令语法，可以发现AGAL支持矩阵的运算。我们将这个过程转移到GPU中去，交给GPU来计算，这样就减轻了CPU的负担。在stage3d_2_变换中提到模型空间、view空间、投影空间的概念。但是之前demo并没有使用到这个，现在在之前的demo基础上，增加模型空间、view空间、投影功能。使得我们可以模拟一个正常的3D环境。最终通过一些列的变换，坐标点数据就会被转换为符合GPU要求的数据。流程如下:
坐标点 -> 转换到模型空间 -> 转换到观察者空间 -> 投影到NDC -> 显示。
伪代码如下:
position ModelMatrix ViewMatrix * ProjectionMatrix

模型空间

模型空间其实就代表了模型在世界坐标系中的状态，例如x轴缩放了2倍，y轴向上位移了10。那么一个坐标点乘以模型空间矩阵之后，这个坐标点就会在x轴上面缩放2倍，y轴向上位移了10单位。将所有模型的坐标点都乘以模型的空间，所有的坐标点都会进行这样一个变换。最终的效果就是，我们只需要简单的修改一下模型空间矩阵，就会达到模型在世界坐标系中的各种变换。

view空间

可以这样脑补一下，桌子上放着一个苹果，我们站在桌子面前，我们在前进、后退、转身等等一些行为之后，桌子上面的苹果在我们的视野中就会发生变化，但是实际上苹果还是那个苹果。。。它一点没有变。为了达到这个效果，就引入了view空间的概念，当进行模型空间变化之后，在进行一次view空间变化，如果修改view空间矩阵，那么view看到的所有模型也会发生变化。

stage3d_4_AGAL以及执行流程

AGAL 什么是AGAL

AGAL在之前稍微说了一下，AGAL其实就是Adobe为了方便开发者，弄的一套高级一点的针对显卡指令的语言。opengl以及dx它们有更加高级的类C的着色语言，分别称之为GLSL、HLSL。而AGAL则更加偏类似于汇编。

寄存器

如果写过汇编的就知道寄存器的概念，我不会详细解释。如果不知道的同学，可以将寄存器理解为一个Vector3D类型。一个寄存器有四个组件，这四个组件分别为x、y、z、w。这四个组件都是浮点类型。寄存器又分为很多类型。

属性寄存器va(只读)

这些寄存器存放的VertexBuffer的顶点数据，也就是存放VertexBuffer的段得数据。这里需要注意的是寄存器并不是存储VertexBuffer的所有数据，而且存储VertexBuffer中的某一段数据，为何是某一段？在下面讲执行流程时我就会说到。在baseline模式中(以下均以baseline为基础讲解)，属性寄存器只有8个。分别为va0,va1,va2….va7

顶点常量寄存器vc(只读)

上一节粗略降到显卡的执行流程分为两阶段。这里的顶点常量寄存器就是第一阶段中需要使用到的寄存器。这个寄存器是常量，既然是常量，那么它也是只读的。顶点的常量寄存器很多，一共有128个。分别为vc0,vc1,vc2….vc127。

顶点临时寄存器vt(读写)

顶点临时寄存器就是顶点着色程序中需要用到的临时变量。我们在编写着色语言时肯定是需要用到临时变量。这个时候我们就需要使用到临时变量寄存器。临时变量寄存器一共有8个。分别为vt0,vt1,vt2…vt7

片段常量寄存器fc(只读)

片段常量寄存器是着色程序第二阶段的需要使用到的寄存器。一共有28个寄存器。分别为fc0,fc1,fc2…fc27

片段临时寄存器ft(读写)

片段临时存器是着色程序第二阶段的需要使用到的临时寄存器器，一共有8个，分别为ft0,ft1,ft2…ft7

片段纹理采样器寄存器

纹理采样寄存器对应着我们上传的贴图，我们传入的贴图被放入到了一个特殊的寄存器里面，我们会从这个寄存器里面获取到正确的贴图颜色。一共有8个。分别为fs0,fs1,fs2…fs7。纹理采样器一共8个，也就是说我们以最小的模型来说，假设这个模型是一个三角形。那么这个三角形最多可以运用8张贴图。

可变寄存器(顶点只写，片段只读)

因为着色流程是先执行顶点程序，然后再执行着色程序。那么有时候我们需要从顶点程序传入数据到片段着色程序，在这个时候我们就需要用到可变寄存器。可变寄存器一共有8个，分别为v0,v1,v2,v3…v7。当然我们不可能从片段着色程序里面传入数据到顶点着色程序。因为这个过程本身就是不可逆的。注意：可变寄存器在顶点程序只能写入，在片段着色程序里面只能读取。

顶点输出寄存器op

顶点输出寄存器专门用于顶点程序，接收最终的结果。

片段输出寄存器oc

片段输出寄存器oc接收最终的颜色结果。

记忆方法:顶点着色程序(vertex program)，以vertex开头字母v记忆，属性(attribute)寄存器就为va，临时寄存器(temp)就为vt，常量寄存器(const)就为vc。片段着色程序(fragment program)，以fragment开头字母f记忆，临时寄存器(temp)为ft,常量寄存器(const)为ft，纹理采样寄存器(sample)为fs。可变寄存器(vary)为v。

AGAL有大约30个不同的指令，不同的指令操作的寄存器数量不同。它们都遵循着=>指令代码 目标寄存器 寄存器1 寄存器2 …例如mov指令:


根据描述，我们可以知道，它是将source寄存器的值移到destination寄存器上。例如:”mov vt0, vt1”。意思为：将顶点临时寄存器vt1的值赋值给顶点临时寄存器vt0。”mov ft0, fc1”:将片段常量寄存器fc1的值赋值给片段临时寄存器ft0。
mov指令只是操作两个寄存。有部分指令是操作三个寄存器。例如:
“add ft0, ft0, fc1” => 将ft0的与fc1相加，然后赋值给ft0。等价于=> ft0 = ft0 + fc1
“sub ft0, ft1, ft0” => ft0 = ft1 - ft0
“mul ft0, ft1, fc1” => ft0 = ft1 * fc1
……
其它指令跟这些指令的操作方式一样。agal在编写的时候需要注意一些注意事项，并且还有一些bug，后面我会在写到的时候提到。
### 组件操作
之前说到一个寄存器有四个组件，每个组件都是一个float类型。我们在agal中可以通过”寄存器” + “.” + “组件名称”的方式来访问。例如:
“add ft0.x, ft0.y, fc1.z” => ft0.x = ft0.y + fc1.z
“add ft0.x, ft0.wz, fc1.zy” => ft0.x = ft0.w + fc1.z => 注意当组件不够用时，要么自动补齐要么自动裁剪多余的。

stage3d_2_变换

我们用几何方式描述3D场景中的物体。在游戏开发中我们会对物体进行变换，例如：平移、缩放、旋转。之前我们谈到向量、矩阵。以及特殊的向量矩阵和矩阵之前的数学运算。在游戏开发中，我们可以通过这种数学运算来描述物体的变换。

基本变换

当我们进行3D编程时，我们使用4x4矩阵来描述变换操作。为何会用到4x4矩阵?因为在3D中，一个点具有三个维度，例如A(x, y, z)。但是为何不用3x3矩阵而是用4x4矩阵。之前向量就谈到，向量只有方向和大小。例如向量A(x, y, z)。这里的向量A和之前的点都是使用同一种方式来描述。所以我们就无法区别(x, y, z)到底是向量还是点。因为向量只与大小方向有关，我们在变换时可能会错误的改变向量的方向以及大小，那么这个时候我们就要用一个标识来区分向量和点。因此我们就会增加一项变为(x, y, z, w)。又根据矩阵的定义，因此我们需要使用4x4矩阵。在其它情况下面，例如透视投影、反射等情况，3x3矩阵同样无法满足我们的需求。因此扩充为4x4。

视角

小时候有一篇课文，讲的是一个小朋友照着一个实物画画，其它小朋友画出的都是一样，唯独这个小朋友画出来的不一样。画杨桃课文 之所以出现这种情况，就是因为每个小朋友的位置方向角度都不一样，因此看到的东西也就不一样。我们称这个小朋友为view空间。在3d游戏中，如果把相机看成一个模型，view其实就是相机的模型空间的逆矩阵。为何是逆矩阵？ 因为我们看到的东西都是反向的。

投影空间

每一个人的计算机屏幕都是不一样的，有些是21英寸、有些是23英寸、27英寸等等。我们在3D开发中不可能去适配所有的尺寸。因此我们需要能够用一种方式来适配所有的情况。gpu已经针对这种情况进行了适配，统一将屏幕的宽高设置为[-1, 1]区间。这样无论尺寸大小如果，总是[-1, 1]区间。因此我们需要将最终的模型映射到这个区间上面来。这个过程就称之为投影。这个区间也称之为NDC空间。在投影的时候，我们可以对投影矩阵进行一些改造，例如达到近大远小的效果(透视投影)，或者达到无论多远都是一样大小(正交投影)。

## 逆矩阵
矩阵代数中没有除法运算,但是它定义了一种乘法的逆(inverse)运算。逆矩阵有如下几个特点:
只有𝑛×𝑛矩阵可以做逆运算。 一个𝑛×𝑛的逆矩阵仍然是一个𝑛×𝑛矩阵。
不是所有的𝑛×𝑛矩阵都有逆矩阵 如果存在逆矩阵，那么这个逆矩阵是唯一的。
* 将一个矩阵与它的逆矩阵相乘，那么结果必定为单位矩阵。
逆矩阵在求解矩阵时，非常有用。例如:一个向量矩阵A乘以矩阵B得到向量矩阵C。那么这个时候就可以通过向量矩阵C乘以矩阵B逆矩阵得到向量矩阵A。

向量

在数学中，向量指具有大小和方向的几何对象，可以形象化地表示为带箭头的线段：箭头所指，代表向量的方向；线段长度，代表向量的大小。一个向量可以有多种记法，如记作粗体的字母（a、b、u、v），或在字母顶上加一小箭头→，或在字母下加波浪线~。如果给定向量的起点（A）和终点（B），可将向量记作AB（并于顶上加→）。

几何表示

向量可以用有向线段来表示。有向线段的长度表示向量的大小，向量的大小，也就是向量的长度。长度为0的向量叫做零向量.长度等于1个单位的向量，叫做单位向量。箭头所指的方向表示向量的方向。（若规定线段AB的端点A为起点，B为终点，则线段就具有了从起点A到终点B的方向和长度。

坐标表示

在平面直角坐标系中，设点A=(x1,y1)，点B=(x2,y2)。则向量AB=(x2-x1,y2-y1)。对于多维的空间向量，通过类推得到。向量的模
向量的大小，也就是向量的长度（称为模）。向量a的模记作|a|。向量的模是非负实数。设向量a=(x,y)，|a|=math.sqrt(xx+yy)。

单位向量

模为1的向量，称之为单位向量。某些情况下面，我们为了计算方便，会将模不为1的向量转换为模为1的向量。例如：玩家角色从A点移动到B点。那么只需要计算出A到B的向量，将向量AB转换为模为1，让玩家从当前位置朝着方向AB移动”模 * speed”距离即可。计算方法：设向量AB=(x,y)。则模为1的向量=(x/|a|, y/|a|)。|a|表示向量的模。

向量点积

设向量a=(x1,y1)，向量b=(x2,y2)。则a·b=x1x2 + y1y2。如果两个向量均未单位向量，那么两个向量的点积就得到两个向量的夹角的cos值。如果cos值越大，那么夹角就越小,cos值越小，夹角就越大。例如：a·b=1时，夹角为0，a·b=0时，夹角为90°。利用单位向量的点击可以用来模拟光照等效果。假设一个三维空间，X轴向右，Y轴向上，Z轴向前。一个平面平铺在xz轴上，用一个向量来表示该平面的方向：a=(0, 1, 0)。假设一个灯光方向为(0, -1, 0)。该灯光垂直照向于该平面。因为屏幕是朝上，但是灯光是照向平面，因此灯光的方向永远都是跟平面的方向相反，因此为了计算方便，我们将灯光方向取反->(0, 1, 0)。则a·b=1。夹角为0，说明灯光垂直照向平面，那么这个时候应该是平面受光最多，平面最亮。以256色为例，设平面颜色为rbg(0, 0, 0)黑色。灯光颜色(255, 0, 0)红色。那么我们可以使用最简单的加法法则：(0, 0, 0) + (255, 0, 0) * 1.0 = (255, 0, 0)。这个时候平面就会表现出红光。如果灯光方向为(1, 0, 0)。说明灯光方向与屏幕互相平行。平面是不会受到光照的。经过计算得出：a·b=0。最终屏幕rgb(0, 0, 0)仍然为黑色。

向量叉积

两个向量a和b的叉积axb的结果为c，c为一个新向量，c垂直于a和b。在3d中，向量叉积使用颇多。例如：前一节向量点积中，我们定义屏幕的方向为(0, 1, 0)。如果我们只知道平面其中的三个不重叠的点以及灯光方向，这个时候需要我们计算出平面受到的光照。设三个点分别为A,B,C。那么向量AB=B-A，向量AC=(C-A)。然后根据向量ABxAC即可计算出一条垂直于该平面的向量，那么这条向量就可以用于光照的计算。
